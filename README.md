# AI Product Engineer 年度训练营

这是你的个人年度学习工程。目标不是“看完一批课程”，而是在一年内具备独立设计、开发、评测和上线 AI 产品的能力。

## 第一次打开

不要先研究所有目录，直接从 [START-HERE.md](./START-HERE.md) 开始。
这里会把每周大纲继续拆成可以直接学习的单节教材。

## 年度毕业标准

- [ ] 完成 48 周有效学习记录
- [ ] 完成不少于 120 个小时的动手实践
- [ ] 上线 2 个可以公开演示的 AI 产品
- [ ] 完成 1 个包含 RAG、Agent、MCP 的综合项目
- [ ] 每个正式项目拥有测试集、评测结果、成本记录和复盘
- [ ] 获得不少于 20 条真实用户反馈
- [ ] 完成 3 篇 AI 产品案例拆解
- [ ] 建立一份 AI Product Engineer 求职作品集

## 学习原则

1. 项目驱动：从第 2 周开始写代码，不等“学完理论”。
2. 小步闭环：每周必须产生一个可检查的成果。
3. 证据优先：不能用“感觉学会了”验收，要用代码、测试或讲解证明。
4. 先原生后框架：先理解模型调用和工具协议，再学习 Agent 框架。
5. 产品视角：每项技术都要回答“它解决了谁的什么问题”。
6. 安全边界：情绪、睡眠等健康相关产品不得输出诊断或替代专业服务。

## 每周固定节奏

建议每周累计投入 5～8 小时，不是每天投入 8 小时。平日每次学习
45～90 分钟即可，周末需要连续实践时最多安排 2～3 小时。

| 环节 | 建议时间 | 必须产出 |
| --- | ---: | --- |
| 原理学习 | 1.5 小时 | 一页自己的解释 |
| 编码实践 | 4 小时 | 可运行代码 |
| 测试评测 | 1 小时 | 测试记录 |
| 产品拆解 | 0.5 小时 | 一条产品观察 |
| 复盘 | 1 小时 | 周复盘 |

推荐拆成 5 次完成：

| 时间 | 内容 | 时长 |
| --- | --- | ---: |
| 第 1 天 | 学习一个核心概念 | 45～60 分钟 |
| 第 2 天 | 完成最小代码实验 | 60～90 分钟 |
| 第 3 天 | 继续编码 | 60～90 分钟 |
| 第 4 天 | 测试失败案例 | 45～60 分钟 |
| 周末任选一天 | 完成交付物和复盘 | 1.5～2.5 小时 |

某天忙时可以跳过，不需要用熬夜补齐。验收看最终是否理解和完成实践，
不看每天坐了多久。

## 使用顺序

1. 阅读 [年度路线图](./docs/roadmap.md)。
2. 阅读 [学习与验收规则](./docs/learning-contract.md)。
3. 打开 [当前学习看板](./dashboard.md)。
4. 从 [第 1 周课程](./weeks/week-01.md) 开始。
5. 每周复制 [周复盘模板](./templates/weekly-review.md) 到 `reviews/`。
6. 每完成一项任务，运行 `npm run progress` 检查总体进度。

## 工程目录

```text
ai-product-engineer-year-plan/
├─ docs/                  年度路线、知识地图、学习规则
├─ demo-app/              所有课程共用的 Vue Demo 项目
├─ notes/                 学习过程中产生的真实问答
├─ weeks/                 每周课程和作业
├─ projects/              三个实战项目
├─ templates/             PRD、Prompt、评测和复盘模板
├─ reviews/               你的周复盘
├─ portfolio/             最终作品集材料
├─ scripts/               本地学习辅助脚本
└─ dashboard.md           当前进度入口
```

所有编程实验都在 `demo-app/` 中按编号区分，不为每节课创建零散的原生
HTML Demo。

## 我们的协作方式

每次学习时，你可以直接告诉老师：

> 开始第 1 周，请按课程带我学习，不要一次把答案都给我。

我会依次进行：

1. 讲清本周最少必要知识；
2. 给你一个小任务；
3. 检查你的答案或代码；
4. 追问关键概念，确认不是照抄；
5. 根据结果决定通过、补做或降低难度；
6. 帮你更新学习记录。
