# 第 2 周：第一次调用模型 API

建议用时：本周累计 5～8 小时，每天 45～90 分钟，单日不超过 2～3 小时  
前置条件：第 1 周通过

## 学习目标

- 理解客户端、服务端和模型服务之间的边界；
- 独立发送一次模型请求；
- 不把 API Key 暴露在浏览器代码中；
- 记录响应内容、耗时和错误。

## 实践任务

1. 使用 Node.js 和环境变量读取 API Key。
2. 发送一条最小对话请求。
3. 将模型名称、开始时间、结束时间和响应打印出来。
4. 故意使用错误 Key，观察错误结构。
5. 故意传入缺少必要字段的请求，观察错误结构。

## 安全要求

- API Key 只能放在未提交的环境变量中；
- `.env` 必须进入 `.gitignore`；
- 日志不得打印完整密钥；
- 前端页面不能直接持有服务商密钥。

## 交付物

- 一个可以运行的最小 Node 脚本；
- 一份成功请求记录；
- 两份失败请求记录；
- 一段对“为什么 Key 不能放前端”的解释。

## 验收问题

1. 为什么前端直接调用模型 API 可能泄露密钥？
2. HTTP 状态码和模型返回内容分别解决什么问题？
3. 超时和模型拒答有什么区别？
4. 如果模型服务不可用，产品界面应该如何表现？

本课程的具体代码会在你完成第 1 周后生成，避免现在被参考答案带着走。
