# W07：文档解析、Chunk 与 Metadata 设计

阶段：RAG 工程

## 为什么学这个

RAG 不是把文档丢进向量库，难点是数据工程和可追溯性。

## 本周要构建什么

建立小型知识库导入流程：解析文档、切 chunk、写 metadata、保留来源定位。

## 本周核心能力

- 文档解析
- Chunk 策略
- Metadata
- 来源追踪

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-07/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-07/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-07/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-07/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-07/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] 导入脚本
- [ ] chunk 样例
- [ ] metadata 设计说明

## 面试讲法

讲清 chunk 大小、重叠、来源字段为什么影响回答质量。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
