# W07：RAG 数据准备：文档解析、Chunk 与 Metadata

阶段：阶段 2：RAG 工程

## 为什么这周适合你

RAG 失败很多时候不是模型问题，而是资料切错了、来源丢了、metadata 不够，导致回答无法追溯。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

建立一个小型接口契约知识库导入流程：导入 markdown / json 文档，切 chunk，写 metadata，保留来源位置。

## 前端切入点

你从前端开发痛点切入：字段来自哪个接口、类型是什么、哪个页面用过、来源在哪里。

## 本周必须补的后端基础

- 文件读取
- 数据清洗
- chunk 策略
- metadata 设计
- 导入脚本

## 本周核心能力

- Document Ingestion
- Chunk
- Metadata
- Source Trace
- 接口契约知识库

## 一周节奏（每天 2～3 小时）

- [Day 01：真实场景与边界](../lessons/week-07/day-01.md) — 先判断这个能力在真实产品里解决什么问题，以及前端、服务端、模型分别负责什么。
- [Day 02：最小代码闭环](../lessons/week-07/day-02.md) — 只写一条能跑通的端到端链路，优先把输入、服务端、模型或数据、输出串起来。
- [Day 03：后端补点与可靠性](../lessons/week-07/day-03.md) — 补上这周必须懂的后端基础，并处理超时、校验、权限、日志或失败兜底。
- [Day 04：失败样本与评测](../lessons/week-07/day-04.md) — 不要只看成功演示，构造失败样本，把问题归因到数据、模型、代码或产品边界。
- [Day 05：作品集与面试表达](../lessons/week-07/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 本周代码任务

- [ ] 准备一批模拟接口文档和页面字段说明。
- [ ] 写 ingest 脚本，把文档切成 chunk。
- [ ] 每个 chunk 保留 docId、sourcePath、section、fieldName 等 metadata。

## 本周交付证据

- [ ] 导入脚本
- [ ] chunk 样例
- [ ] metadata 字段说明

## 面试讲法

RAG 不是把文档扔进向量库，我会先设计 chunk 和 metadata，保证答案能追溯到来源。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
