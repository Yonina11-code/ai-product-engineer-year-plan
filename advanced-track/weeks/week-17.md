# W17：MCP + RAG + Tool 综合助手

阶段：MCP

## 为什么学这个

这一周把前面的能力串起来，形成一个够硬的综合案例。

## 本周要构建什么

让助手通过 MCP 查询契约、通过 RAG 找文档、通过工具生成开发清单。

## 本周核心能力

- MCP 集成
- RAG 引用
- 多工具编排
- Trace

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-17/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-17/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-17/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-17/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-17/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] 综合 Demo
- [ ] 多工具 Trace
- [ ] 失败样本

## 面试讲法

讲清一个复杂 AI 功能的端到端架构。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
