# W04：Prompt Registry、版本管理与灰度

阶段：AI 应用工程底座

## 为什么学这个

你不需要学“Prompt 不同输出不同”，你需要学 Prompt 如何像工程配置一样被管理。

## 本周要构建什么

实现 Prompt Registry：按 taskType 选择模板，记录版本、变更原因和回滚策略。

## 本周核心能力

- Prompt 版本
- 任务路由
- 配置与代码分离
- 灰度/回滚

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-04/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-04/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-04/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-04/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-04/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] Prompt Registry 代码
- [ ] 版本变更记录
- [ ] 一次回归对比

## 面试讲法

讲清为什么 Prompt 不是散落在代码里的字符串。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
