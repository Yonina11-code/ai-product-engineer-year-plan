# W12：Tool Calling：模型建议，程序裁决

阶段：阶段 3：Tool / Agent / MCP

## 为什么这周适合你

Tool Calling 不是让模型直接执行函数，而是让模型提出工具调用意图，再由程序做参数校验、权限判断和执行。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

给接口契约助手增加两个只读工具：lookupApiField 和 searchCodeReference，工具参数必须过 Schema。

## 前端切入点

前端展示工具调用 Trace：模型为什么选这个工具、传了什么参数、工具返回了什么。

## 本周必须补的后端基础

- Tool Schema
- 参数校验
- 只读工具
- 工具结果归一化
- Trace 记录

## 本周核心能力

- Tool Calling
- Tool Schema
- 参数校验
- 只读工具

## 一周节奏（每天 2～3 小时）

- [Day 05：作品集与面试表达](../lessons/week-12/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 面试讲法

我不会直接相信模型生成的工具参数，工具执行前必须经过 Schema、权限和业务规则校验。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
