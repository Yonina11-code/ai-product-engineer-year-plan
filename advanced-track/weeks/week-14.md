# W14：Workflow vs Agent：状态机、Trace 与终止条件

阶段：工具调用与 Agent

## 为什么学这个

企业更关心可控性。你要知道什么时候用固定 Workflow，什么时候才用 Agent。

## 本周要构建什么

同一个任务分别实现 Workflow 和 Agent 版本，对比可测试性和失败边界。

## 本周核心能力

- Workflow
- Agent Loop
- Trace
- 终止条件

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-14/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-14/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-14/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-14/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-14/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] 两种实现对比
- [ ] Trace 日志
- [ ] 死循环防护

## 面试讲法

讲清 Agent 不是越自由越好。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
