# W14：Workflow vs Agent：什么时候不该用 Agent

阶段：阶段 3：Tool / Agent / MCP

## 为什么这周适合你

很多任务用固定 workflow 更稳定。只有当下一步依赖中间结果、流程不固定时，才考虑 Agent。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

同一个“生成开发清单”任务分别做 workflow 版和 agent loop 版，对比可控性、可测试性和失败边界。

## 前端切入点

前端展示两种模式的执行步骤，让你直观看到固定流程和 Agent 动态决策的差异。

## 一周节奏（每天 2～3 小时）

- [Day 01：真实场景与边界](../lessons/week-14/day-01.md) — 先判断这个能力在真实产品里解决什么问题，以及前端、服务端、模型分别负责什么。
- [Day 02：最小代码闭环](../lessons/week-14/day-02.md) — 只写一条能跑通的端到端链路，优先把输入、服务端、模型或数据、输出串起来。
- [Day 03：后端补点与可靠性](../lessons/week-14/day-03.md) — 补上这周必须懂的后端基础，并处理超时、校验、权限、日志或失败兜底。
- [Day 04：失败样本与评测](../lessons/week-14/day-04.md) — 不要只看成功演示，构造失败样本，把问题归因到数据、模型、代码或产品边界。
- [Day 05：作品集与面试表达](../lessons/week-14/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 本周代码任务

- [ ] 实现固定 workflow：检索 -> 生成清单 -> 校验 -> 输出。
- [ ] 实现受限 Agent：最多 5 步，只能调用只读工具。
- [ ] 对比两种模式的失败样本和可测试性。

## 本周交付证据

- [ ] 两种实现对比
- [ ] Agent Trace 日志
- [ ] 死循环防护案例

## 面试讲法

我不会为了炫技乱用 Agent，能写死流程就用 workflow，只有流程依赖中间结果时才用受限 Agent。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
