# W09：Query Rewrite、Rerank 与上下文选择

阶段：RAG 工程

## 为什么学这个

真实用户不会按你的文档字段名提问，所以要处理口语化、缩写和多意图。

## 本周要构建什么

实现 Query Rewrite、候选文档重排和上下文预算控制。

## 本周核心能力

- 查询改写
- Rerank
- 上下文预算
- 多路召回合并

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-09/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-09/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-09/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-09/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-09/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] 改写前后召回对比
- [ ] Rerank Trace
- [ ] 上下文裁剪规则

## 面试讲法

讲清如何提升 RAG 命中率而不是盲目加更多文档。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
