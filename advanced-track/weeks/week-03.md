# W03：结构化输出、Schema 与自动修复

阶段：AI 应用工程底座

## 为什么学这个

找工作时，“会调模型”不够，必须证明你能让模型输出进入业务系统。

## 本周要构建什么

设计 JSON 输出契约、运行时校验、失败重试和一次自动修复流程。

## 本周核心能力

- JSON Schema
- 运行时校验
- Repair Prompt
- 业务字段契约

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-03/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-03/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-03/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-03/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-03/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] Schema 文件
- [ ] 10 条格式失败样本
- [ ] 修复前后对照

## 面试讲法

讲清格式正确、内容合格、业务成功三者不是一回事。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
