# W01：真实模型 Gateway 与调用边界

阶段：AI 应用工程底座

## 为什么学这个

你已经会写前端和接口，这周直接做真实模型调用的工程边界，而不是再讲 Prompt 是什么。

## 本周要构建什么

在 demo-app 中完成一个可切换 Provider 的 AI Gateway：mock / DeepSeek，支持超时、错误归类、请求日志和敏感配置隔离。

## 本周核心能力

- Provider 抽象
- 服务端 API Key 管理
- 超时与错误分类
- 模型响应归一化

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-01/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-01/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-01/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-01/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-01/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] 一张 AI Gateway 架构图
- [ ] 一次真实模型调用记录
- [ ] 三类失败错误截图或日志

## 面试讲法

如果面试官问“你怎么把大模型接进现有前端系统”，你要能讲清为什么不在前端直接调模型。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
