# W01：AI Gateway：前端如何安全接入真实模型

阶段：阶段 1：AI 应用工程底座

## 为什么这周适合你

前端不能直接把 API Key 暴露给浏览器，也不能把模型错误、超时、供应商差异全部丢给页面处理。本周你要做的是 AI 应用的服务端入口。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

在 demo-app 里做一个 Node AI Gateway：前端只请求自己的服务端，服务端支持 mock / DeepSeek Provider，统一返回格式、错误类型和 requestId。

## 前端切入点

用你熟悉的 Vue 页面发起请求、展示 loading / success / error，并观察 Network 里前端到底看不到哪些敏感信息。

## 本周必须补的后端基础

- Node HTTP 路由
- 环境变量读取
- Provider service 分层
- 请求超时
- 错误分类

## 本周核心能力

- AI Gateway
- Provider 抽象
- API Key 服务端隔离
- 统一响应契约
- requestId

## 一周节奏（每天 2～3 小时）

- [Day 01：真实场景与边界](../lessons/week-01/day-01.md) — 先判断这个能力在真实产品里解决什么问题，以及前端、服务端、模型分别负责什么。
- [Day 02：最小代码闭环](../lessons/week-01/day-02.md) — 只写一条能跑通的端到端链路，优先把输入、服务端、模型或数据、输出串起来。
- [Day 03：后端补点与可靠性](../lessons/week-01/day-03.md) — 补上这周必须懂的后端基础，并处理超时、校验、权限、日志或失败兜底。
- [Day 04：失败样本与评测](../lessons/week-01/day-04.md) — 不要只看成功演示，构造失败样本，把问题归因到数据、模型、代码或产品边界。
- [Day 05：作品集与面试表达](../lessons/week-01/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 本周代码任务

- [ ] 新增或整理 /api/ai/chat 接口，前端只调用自己的服务端。
- [ ] 实现 mockProvider 和 deepseekProvider 的统一接口。
- [ ] 给请求加 requestId、timeoutMs、错误类型和用户可见错误文案。

## 本周交付证据

- [ ] 一张 AI Gateway 边界图
- [ ] 一次真实 DeepSeek 调用日志
- [ ] 三类失败日志：无 Key、超时、Provider 错误

## 面试讲法

我没有在前端直接调模型，而是在服务端做 Gateway，隔离 API Key，并统一 Provider、错误、日志和降级策略。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
