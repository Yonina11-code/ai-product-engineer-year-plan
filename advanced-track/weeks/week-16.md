# W16：项目三：MCP + RAG + Tool 综合接口助手

阶段：阶段 3 项目验收

## 为什么这周适合你

这一周把 RAG、Tool、Agent 和 MCP 串成一个真正能讲清楚的综合项目。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

让助手通过 RAG 找证据，通过 Tool 查询结构化字段，通过受限 Agent 决定是否继续查，并把工具封装成 MCP。

## 前端切入点

前端重点展示完整 Trace：检索了什么、调用了什么工具、为什么拒答或继续查。

## 本周必须补的后端基础

- 多能力编排
- Trace 数据结构
- 错误传播
- 权限分层
- 综合项目验收

## 本周核心能力

- RAG + Tool
- 受限 Agent
- MCP 封装
- Trace
- 综合架构

## 一周节奏（每天 2～3 小时）

- [Day 01：真实场景与边界](../lessons/week-16/day-01.md) — 先判断这个能力在真实产品里解决什么问题，以及前端、服务端、模型分别负责什么。
- [Day 02：最小代码闭环](../lessons/week-16/day-02.md) — 只写一条能跑通的端到端链路，优先把输入、服务端、模型或数据、输出串起来。
- [Day 03：后端补点与可靠性](../lessons/week-16/day-03.md) — 补上这周必须懂的后端基础，并处理超时、校验、权限、日志或失败兜底。
- [Day 04：失败样本与评测](../lessons/week-16/day-04.md) — 不要只看成功演示，构造失败样本，把问题归因到数据、模型、代码或产品边界。
- [Day 05：作品集与面试表达](../lessons/week-16/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 面试讲法

我能讲清一个 AI 系统里模型、RAG、Tool、Agent、MCP 分别解决什么问题，以及为什么不混用。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
