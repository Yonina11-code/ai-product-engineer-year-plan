# W16：MCP Server：把现有能力封装给 AI 使用

阶段：MCP

## 为什么学这个

MCP 的价值不是名词，而是把工具、资源和权限以标准方式暴露给模型客户端。

## 本周要构建什么

用 Node 写一个最小 MCP Server，封装一个只读接口契约查询工具。

## 本周核心能力

- MCP Tool
- MCP Resource
- Schema
- 权限边界

## 一周节奏（每天 2～3 小时）

- [Day 01：架构判断](../lessons/week-16/day-01.md) — 读代码、画边界、明确这项能力解决什么工程问题。
- [Day 02：最小实现](../lessons/week-16/day-02.md) — 写一个能跑的薄切片，不追求大而全。
- [Day 03：可靠性增强](../lessons/week-16/day-03.md) — 补错误处理、结构校验、边界条件和可观测信息。
- [Day 04：评测与失败分析](../lessons/week-16/day-04.md) — 构造样本集，记录失败原因，避免只看成功演示。
- [Day 05：作品集与面试表达](../lessons/week-16/day-05.md) — 沉淀架构图、代码证据、指标和三分钟讲法。

## 本周交付证据

- [ ] 最小 MCP Server
- [ ] 工具调用截图
- [ ] 错误参数拒绝记录

## 面试讲法

讲清 MCP 和普通后端 API 的关系。

## 通过标准

- 能跑出最小闭环；
- 有失败样本和处理策略；
- 有日志、截图、测试或评测证据；
- 能用自己的项目解释这项能力；
- 不把概念讲成空话。
