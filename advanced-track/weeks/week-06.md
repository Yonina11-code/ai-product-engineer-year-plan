# W06：项目一：AI 职场沟通助手工程化版本

阶段：阶段 1 项目验收

## 为什么这周适合你

这个项目不是练 Prompt，而是证明你能把一个 AI 文本能力做成可用、可测、可复盘的小产品。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

完成职场沟通助手 MVP：真实模型、结构化输出、Prompt Registry、评测集、日志、复制结果、失败降级。

## 前端切入点

做一个像真实功能的页面：输入原话、选择目标语气、展示改写结果、解释改写原因、复制输出。

## 本周必须补的后端基础

- 端到端接口设计
- DTO / ViewModel 边界
- 错误码到 UI 状态映射
- 最小验收接口
- 本地部署

## 本周核心能力

- 端到端闭环
- 产品交互
- 评测证据
- 失败兜底
- 项目表达

## 一周节奏（每天 2～3 小时）

- [Day 01：真实场景与边界](../lessons/week-06/day-01.md) — 先判断这个能力在真实产品里解决什么问题，以及前端、服务端、模型分别负责什么。
- [Day 02：最小代码闭环](../lessons/week-06/day-02.md) — 只写一条能跑通的端到端链路，优先把输入、服务端、模型或数据、输出串起来。
- [Day 03：后端补点与可靠性](../lessons/week-06/day-03.md) — 补上这周必须懂的后端基础，并处理超时、校验、权限、日志或失败兜底。
- [Day 04：失败样本与评测](../lessons/week-06/day-04.md) — 不要只看成功演示，构造失败样本，把问题归因到数据、模型、代码或产品边界。
- [Day 05：作品集与面试表达](../lessons/week-06/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 本周代码任务

- [ ] 把前 5 周能力收敛到一个完整页面和一个清晰 API。
- [ ] 补齐 30 条评测样本和失败样本。
- [ ] 整理 README：如何运行、如何验证、有什么边界。

## 本周交付证据

- [ ] 可演示项目
- [ ] 30 条评测结果
- [ ] 项目 README 与架构图

## 面试讲法

我做的是一个可上线雏形：前端体验、服务端 Gateway、结构化校验、Prompt 版本、评测和日志都有闭环。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
