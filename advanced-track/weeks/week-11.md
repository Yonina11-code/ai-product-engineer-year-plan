# W11：项目二：前端接口契约 RAG 助手

阶段：阶段 2 项目验收

## 为什么这周适合你

这是最贴合你前端背景的核心项目：把你对字段契约、接口联调、页面排查的经验变成 AI 助手。

你是有经验的前端，所以本周不会停留在概念解释，而是从“一个真实 AI 功能如何进入现有前端系统”开始。

## 本周要构建什么

完成一个接口契约 RAG 助手：能查字段来源、类型、枚举、页面使用点；不能确定时拒答或追问。

## 前端切入点

做一个真实可演示的查询页面，展示答案、引用、未知字段、相关页面和失败原因。

## 本周必须补的后端基础

- RAG API 分层
- 知识库更新流程
- 缓存策略
- 权限边界
- 项目 README

## 本周核心能力

- 接口契约助手
- RAG 闭环
- 字段来源
- 引用定位
- 未知追问

## 一周节奏（每天 2～3 小时）

- [Day 01：真实场景与边界](../lessons/week-11/day-01.md) — 先判断这个能力在真实产品里解决什么问题，以及前端、服务端、模型分别负责什么。
- [Day 02：最小代码闭环](../lessons/week-11/day-02.md) — 只写一条能跑通的端到端链路，优先把输入、服务端、模型或数据、输出串起来。
- [Day 03：后端补点与可靠性](../lessons/week-11/day-03.md) — 补上这周必须懂的后端基础，并处理超时、校验、权限、日志或失败兜底。
- [Day 04：失败样本与评测](../lessons/week-11/day-04.md) — 不要只看成功演示，构造失败样本，把问题归因到数据、模型、代码或产品边界。
- [Day 05：作品集与面试表达](../lessons/week-11/day-05.md) — 整理架构图、关键代码、日志、评测结果和一段能被面试追问的讲法。

## 本周代码任务

- [ ] 把 W7-W10 能力收敛成一个完整项目。
- [ ] 补齐 50 条评测与 5 个失败案例。
- [ ] 整理项目故事：为什么这是前端转 AI 的优势项目。

## 本周交付证据

- [ ] 可演示 RAG 项目
- [ ] 50 条评测报告
- [ ] 字段引用截图与架构图

## 面试讲法

这个项目证明我不是只会调模型，而是能把 AI 用在前端真实工程痛点：接口字段、契约和排查效率。

## 通过标准

- 能跑出端到端最小闭环；
- 能说清前端、服务端、模型、数据各自边界；
- 有失败样本和处理策略；
- 有日志、评测、截图、Trace 或架构图证据；
- 能把本周能力写进作品集项目，而不是只背概念。
