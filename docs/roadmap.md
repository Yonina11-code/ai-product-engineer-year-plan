# 高级求职向路线图

这份路线图替换原来的 12 个月入门路线。

原因：你已经是 6～7 年前端工程师，不需要再按“认识 Prompt、认识 AI 应用链路”的节奏慢慢学。新的路线直接服务于找工作。

## 总长度

24 周，每天 2～3 小时。

可以压缩，也可以拉长。目标不是把周数刷满，而是拿到足够硬的求职项目证据。

## 阶段 1：AI 应用工程底座

目标：你能把大模型安全、稳定、可观测地接入现有前端/服务端系统。

- 真实模型 Gateway 与调用边界
- Streaming、取消生成与体验状态机
- 结构化输出、Schema 与自动修复
- Prompt Registry、版本管理与灰度
- 评测集、回归测试与质量门禁
- 项目一：AI 职场沟通助手工程化重做

阶段验收：

- [ ] 可切换 Provider 的 AI Gateway
- [ ] 真实模型调用日志
- [ ] 结构化输出校验和失败修复
- [ ] 30 条评测集
- [ ] 一个能演示的 AI 功能

## 阶段 2：RAG 工程

目标：你能构建有来源、可评测、不乱编字段的知识助手。

- 文档解析、Chunk 与 Metadata 设计
- Embedding、向量检索与混合召回
- Query Rewrite、Rerank 与上下文选择
- 引用、拒答与可信回答
- RAG 评测：命中率、正确性与引用质量
- 项目二：前端接口契约 RAG 助手

阶段验收：

- [ ] 导入和检索流程
- [ ] 50 条 RAG 评测集
- [ ] 引用来源和拒答机制
- [ ] 一个贴合前端经验的 RAG 项目

## 阶段 3：Tool / Agent / MCP

目标：你能让模型安全调用工具，并能解释 Agent 的可靠性边界。

- Tool Calling：只读工具、写工具与权限边界
- Workflow vs Agent：状态机、Trace 与终止条件
- Agent 可靠性：幂等、重试、人工接管
- MCP Server：把现有能力封装给 AI 使用
- MCP + RAG + Tool 综合助手

阶段验收：

- [ ] 至少两个工具定义
- [ ] 写操作确认流程
- [ ] Agent Trace 与终止条件
- [ ] 最小 MCP Server
- [ ] 综合 Demo

## 阶段 4：产品化、上线与求职

目标：把能力转成作品集、简历和面试表达。

- 可观测性：日志、Token、成本、延迟与质量
- 安全、隐私与高风险边界
- 项目三：AI 睡眠/情绪教练 MVP
- 作品集工程化整理
- AI 工程面试专项
- 简历、岗位匹配与投递材料
- 最终答辩与下一阶段计划

阶段验收：

- [ ] 作品集首页
- [ ] 三个项目的架构图和评测证据
- [ ] AI 工程面试问答
- [ ] 两版简历项目描述
- [ ] 一次 45 分钟模拟答辩
