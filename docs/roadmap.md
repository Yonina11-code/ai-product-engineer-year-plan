# 12 个月路线图

## 第一季度：模型应用与 Prompt

目标：独立完成一个可靠的小型 AI 功能。

### 第 1 月：认识模型与 API

- AI 应用的请求链路
- Token、上下文窗口、Temperature
- System、User、Assistant 消息
- 流式输出
- 模型能力边界和幻觉

### 第 2 月：Prompt Engineering

- 任务、上下文、约束、示例、输出格式
- Zero-shot 与 Few-shot
- Prompt 版本管理
- 失败样本分析
- Prompt 测试集

### 第 3 月：可靠输出

- JSON 与结构化输出
- 数据校验
- 超时、重试和降级
- 延迟、Token 与成本记录
- 完成项目一 MVP

季度验收：

- [ ] 30 条 Prompt 测试集
- [ ] 一个已部署的职场沟通助手
- [ ] 五条真实用户反馈
- [ ] 一篇项目复盘

## 第二季度：Agent 与 MCP

目标：让模型安全、可控地调用工具完成任务。

### 第 4 月：工具调用

- Function Calling
- JSON Schema
- 参数校验
- 工具错误处理
- 用户确认机制

### 第 5 月：Agent Workflow

- Agent 与固定 Workflow 的区别
- 任务拆解和状态管理
- 循环终止条件
- 重试与幂等
- 可观测日志

### 第 6 月：MCP

- Client、Server、Tool、Resource
- 使用现有 MCP Server
- 编写一个最小 MCP Server
- 权限和数据安全
- 将一个明确接口封装为工具

季度验收：

- [ ] 三个可调用工具
- [ ] 一个最小 MCP Server
- [ ] 十条工具调用异常测试
- [ ] 一份 Agent 安全边界说明

## 第三季度：RAG 与知识产品

目标：构建有来源、可评测、不乱编字段的知识助手。

### 第 7 月：RAG 基础

- 文档解析
- Chunk 切分
- Embedding
- 向量召回
- 引用来源

### 第 8 月：检索质量

- 关键词与向量混合检索
- Metadata Filter
- Query Rewrite
- Rerank
- 无答案拒答

### 第 9 月：RAG 评测

- 构造问题集
- 检索命中率
- 答案正确性
- 引用正确性
- 完成项目二

季度验收：

- [ ] 50 条 RAG 评测集
- [ ] 每条答案可以追溯来源
- [ ] 未知字段会询问而不是猜测
- [ ] 项目二可在线演示

## 第四季度：产品化与转型

目标：从功能开发者成长为能做完整产品闭环的 AI Product Engineer。

### 第 10 月：产品设计

- 用户访谈
- 问题定义
- AI 必要性判断
- 产品指标
- 安全与隐私

### 第 11 月：综合项目

- AI 睡眠教练 MVP
- 日志、计划、周总结
- 个性化但非医疗化建议
- 数据删除和隐私
- 真实用户测试

### 第 12 月：作品集与面试

- 整理架构图和演示视频
- 补充评测与成本数据
- 撰写项目复盘
- 准备 AI Product Engineer 简历
- 模拟技术与产品面试

年度验收：

- [ ] 两个正式上线项目
- [ ] 一个综合项目
- [ ] 作品集首页
- [ ] 三分钟个人项目介绍
- [ ] 一轮模拟面试
