# 前端老手转 AI 应用工程路线图

这份路线图替换原来的入门路线。

原因：你已经是 6～7 年经验前端工程师，不需要再按“认识 Prompt、认识 AI 应用链路”的节奏慢慢学。你当前真正缺的是实战闭环：前端怎么接真实模型、哪些能力放服务端、RAG / Tool / Agent / MCP 怎么串、怎么评测和上线、怎么形成求职项目证据。

## 总长度

20 周，每天 2～3 小时。

可以压缩，也可以拉长。目标不是把周数刷满，而是拿到足够硬的求职项目证据。

## 阶段 1：AI 应用工程底座

目标：你能把大模型安全、稳定、可观测地接入现有前端 + 服务端系统。

- W01：AI Gateway：前端如何安全接入真实模型
- W02：Streaming：AI 响应慢时前端体验怎么做
- W03：结构化输出：让模型结果能进入业务系统
- W04：Prompt Registry 与评测：不要把 Prompt 写成散落字符串
- W05：会话、日志、成本：AI 功能上线前必须补的后端基础
- W06：项目一：AI 职场沟通助手工程化版本

阶段验收：

- [ ] 前端不直接暴露 API Key；
- [ ] 服务端有 AI Gateway、Provider 抽象、错误分类；
- [ ] 支持 Streaming、取消和重试；
- [ ] 模型输出经过 Schema 校验；
- [ ] 有 Prompt 版本、评测集、调用日志和成本记录；
- [ ] AI 职场沟通助手可以演示并能被面试追问。

## 阶段 2：RAG 工程

目标：你能构建有来源、可评测、不乱编字段的知识助手。

- W07：RAG 数据准备：文档解析、Chunk 与 Metadata
- W08：检索实现：关键词、向量与混合召回
- W09：可信回答：引用、拒答与冲突证据
- W10：RAG 评测：把“效果还行”变成指标
- W11：项目二：前端接口契约 RAG 助手

阶段验收：

- [ ] 有文档导入、chunk 和 metadata 设计；
- [ ] 能比较关键词、向量和混合召回；
- [ ] 回答带引用，证据不足时拒答；
- [ ] 有 50 条 RAG 评测集；
- [ ] 前端接口契约 RAG 助手可以演示。

## 阶段 3：Tool / Agent / MCP

目标：你能让模型安全调用工具，并能解释什么时候用 Workflow、Agent、MCP。

- W12：Tool Calling：模型建议，程序裁决
- W13：写操作边界：确认、幂等与审计
- W14：Workflow vs Agent：什么时候不该用 Agent
- W15：MCP Server：把你的工具标准化暴露给 AI
- W16：项目三：MCP + RAG + Tool 综合接口助手

阶段验收：

- [ ] 至少两个只读工具定义；
- [ ] 写操作有二次确认、幂等和审计；
- [ ] 能比较 Workflow 和 Agent 的取舍；
- [ ] 有最小 MCP Server；
- [ ] 综合项目有完整 Trace 和失败样本。

## 阶段 4：AI 产品边界与安全

目标：你能做偏产品方向的 AI MVP，并处理健康、情绪类场景的风险边界。

- W17：AI 产品边界：睡眠 / 情绪助手为什么难
- W18：项目四：AI 睡眠 / 情绪教练 MVP

阶段验收：

- [ ] 明确目标用户和非目标用户；
- [ ] 不做医疗诊断，不替代专业建议；
- [ ] 有高风险输入识别和安全提示；
- [ ] 有用户反馈和失败样本；
- [ ] AI 睡眠 / 情绪教练 MVP 可以演示。

## 阶段 5：上线、作品集与求职

目标：把能力转成作品集、简历和面试表达。

- W19：部署、监控与成本：作品集也要像真实项目
- W20：作品集、简历与 45 分钟项目答辩

阶段验收：

- [ ] 项目有部署说明和环境变量说明；
- [ ] 有延迟、错误率、token、成本和质量样本；
- [ ] 作品集首页完成；
- [ ] 三个核心项目有架构图和评测证据；
- [ ] 有两版简历项目描述；
- [ ] 完成一次 45 分钟模拟答辩。
