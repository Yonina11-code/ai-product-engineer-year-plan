# 学习与验收契约

## 我负责什么

- 根据你的实际进度备课，而不是机械照着目录念。
- 用前端开发经验连接 AI 概念，减少无意义的数学和训练细节。
- 为每项知识安排可以运行的小实验。
- 检查代码、Prompt、评测集和产品设计。
- 当你没有真正理解时明确指出，不用“完成了”安慰你。
- 根据工作忙闲调整周任务，但不取消实践和复盘。

## 你负责什么

- 每周争取累计投入 5～8 小时，单次通常 45～90 分钟，单日不超过
  2～3 小时。
- 先尝试，再请求完整答案。
- 如实记录卡点、失败结果和实际用时。
- 所有作业都能运行或展示。
- 每周至少提出一个自己的问题。
- 不用购买课程或收集资料代替动手。

## 掌握等级

| 等级 | 表现 |
| --- | --- |
| 0 未接触 | 不知道概念解决什么问题 |
| 1 听懂 | 能跟随解释，但不能独立完成 |
| 2 会模仿 | 能参考示例完成同类任务 |
| 3 会应用 | 能独立在新场景中使用 |
| 4 会判断 | 能解释取舍、风险和替代方案 |
| 5 会教学 | 能让别人理解并完成实践 |

年度目标不是所有知识达到 5 级。核心能力达到 3～4 级即可：

- Prompt 与评测：4
- 模型 API 与错误处理：4
- RAG：3
- Agent 与工具调用：3
- MCP：3
- AI 产品设计：3
- 模型训练原理：1～2

## 每周通过流程

1. 自测：完成课程中的交付物。
2. 讲解：用自己的语言回答验收问题。
3. 变式：老师修改一个条件，你独立调整实现。
4. 边界：补充至少一个失败或异常测试。
5. 复盘：记录学会、卡点和下周行动。

## 延期处理

- 工作繁忙：缩小任务，但保留一个可交付结果。
- 当周只能投入 2～3 小时：切换为轻量周，只完成一个概念、一个实验和
  一次简短复盘。
- 连续两周未完成：停止新内容，恢复到最小学习节奏。
- 某概念连续两次未通过：换一种解释和练习，不重复硬背。
- 项目过大：砍功能，不延长没有边界的开发周期。
