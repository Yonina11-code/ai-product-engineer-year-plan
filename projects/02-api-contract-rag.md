# 项目二：AI 接口契约与需求助手

周期：第 25～36 周  
项目定位：体现你已有前端工程经验的差异化作品

## 要解决的问题

前端开发面对需求文档、接口文档和历史代码时，容易因字段来源不清、类型不一致或缺少依据而反复沟通。

## 核心能力

- [ ] 导入明确范围内的需求和接口文档
- [ ] 根据问题检索相关原文
- [ ] 回答时展示引用来源
- [ ] 提取字段名、类型、是否必填和说明
- [ ] 字段不明确时明确说“不确定”
- [ ] 一次只追问一个待确认字段
- [ ] 生成前端对接检查清单

## 强制边界

- 不从多个未知字段兜底取值；
- 不根据相似代码猜测接口字段；
- 不擅自重命名或转换字段类型；
- 详情回显以详情接口原始字段为准；
- 所有结论必须标注来源；
- 没有来源时必须要求人工确认。

## 技术能力覆盖

- 文档解析和 Chunk
- Embedding
- 混合检索
- Metadata Filter
- Rerank
- 引用生成
- RAG 评测
- MCP 工具封装

## 毕业标准

- [ ] 50 条真实或脱敏问题集
- [ ] 字段回答引用正确率达到预设目标
- [ ] 对知识库外问题能够拒答
- [ ] 至少十条模糊字段测试
- [ ] 提供架构图、演示和复盘
