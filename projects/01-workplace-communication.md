# 项目一：AI 职场沟通助手

周期：第 5～12 周  
项目定位：第一个可上线、可评测的 AI 产品

## 核心用户

需要在工作中表达不同意见，但担心措辞生硬、模糊或过度客气的职场人士。

## 核心任务

输入原始表达、沟通对象、沟通目标和期望语气，输出保持原意的专业表达。

## MVP 范围

- [ ] 输入原始消息
- [ ] 选择沟通对象
- [ ] 选择语气
- [ ] 填写沟通目标
- [ ] 生成一个主要版本
- [ ] 展示修改说明
- [ ] 复制结果
- [ ] 对结果标记满意或不满意

## 暂不开发

- 登录和复杂用户体系
- 多 Agent
- 语音输入
- 企业通讯录
- 自动发送消息
- 十几种模型切换

## 可靠性要求

- 不虚构事实；
- 不改变用户立场；
- 不增加未经输入的承诺；
- 不输出攻击、歧视或威胁内容；
- 模型失败时保留用户输入；
- 结构化结果校验失败时给出明确提示。

## 评测维度

| 维度 | 判断问题 |
| --- | --- |
| 原意保持 | 是否改变核心立场或事实？ |
| 专业程度 | 是否适合真实工作沟通？ |
| 简洁程度 | 是否出现空话和重复？ |
| 语气匹配 | 是否符合用户选择？ |
| 可直接使用 | 用户是否愿意复制发送？ |

## 毕业标准

- [ ] 30 条测试用例
- [ ] 两个 Prompt 版本对比
- [ ] 五名真实用户试用
- [ ] 记录延迟和单次成本
- [ ] 部署到可访问环境
- [ ] 完成项目复盘
