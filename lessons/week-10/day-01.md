# 第 10 周 · 第 1 天：建立概念模型

本周主题：结构化输出与 JSON Schema

所属模块：可靠输出

建议时间：45～90 分钟

## 今日目标

建立JSON Output、Schema的正确直觉。

## 1. 从问题出发

本周要解决的工程问题是：**结构化输出与 JSON Schema怎样让 AI 应用更可靠或更有价值？**

场景采用：DeepSeek 输出业务 JSON 后进入 Vue 页面前的校验层。

模型结果必须像不可信的第三方接口数据一样经过运行时校验，TypeScript 类型本身不能保护运行时。

## 2. 今天的核心知识

### 1. JSON Output

JSON Output 让自然语言结果进入业务代码前具有明确结构，但“能解析 JSON”不等于字段正确或内容可信。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

### 2. Schema

Schema 明确定义字段、类型和约束，用于验证运行时模型输出。它是模型结果与业务代码之间的契约。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

## 3. 画出位置

请把本周能力放进下面链路：

```text
用户问题 → 前端输入 → 服务端规则 → 模型/检索/工具
→ 结果校验 → 产品状态 → 用户反馈
```

重点标记：Schema、必填字段和错误分支是确定性代码；模型是否按要求生成仍有不确定性。

## 4. 今日练习

用不超过200字解释本周主题，要求同时出现：

- 一个真实使用场景；
- 一个确定性程序职责；
- 一个模型不确定性；
- 一个失败风险。
