# 第 11 周 · 第 1 天：建立概念模型

本周主题：运行时校验与自动修复

所属模块：可靠输出

建议时间：45～90 分钟

## 今日目标

建立类型校验、语义校验的正确直觉。

## 1. 从问题出发

本周要解决的工程问题是：**运行时校验与自动修复怎样让 AI 应用更可靠或更有价值？**

场景采用：DeepSeek 输出业务 JSON 后进入 Vue 页面前的校验层。

模型结果必须像不可信的第三方接口数据一样经过运行时校验，TypeScript 类型本身不能保护运行时。

## 2. 今天的核心知识

### 1. 类型校验

类型校验 是本周链路中的一个关键决策点。理解时不要只背定义，要在“DeepSeek 输出业务 JSON 后进入 Vue 页面前的校验层”中确认它接收什么输入、产生什么输出、失败时由谁负责。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

### 2. 语义校验

语义校验 是本周链路中的一个关键决策点。理解时不要只背定义，要在“DeepSeek 输出业务 JSON 后进入 Vue 页面前的校验层”中确认它接收什么输入、产生什么输出、失败时由谁负责。

学习后请回答：

- 哪种情况下不应该使用？

## 3. 画出位置

请把本周能力放进下面链路：

```text
用户问题 → 前端输入 → 服务端规则 → 模型/检索/工具
→ 结果校验 → 产品状态 → 用户反馈
```

重点标记：Schema、必填字段和错误分支是确定性代码；模型是否按要求生成仍有不确定性。

## 4. 今日练习

用不超过200字解释本周主题，要求同时出现：

- 一个真实使用场景；
- 一个确定性程序职责；
- 一个模型不确定性；
- 一个失败风险。

- 能指出至少一个风险或限制；
- 没有用模型替代本应由程序控制的明确规则；
