# 第2课：大模型为什么会不稳定

建议时间：45～60 分钟  
前置条件：第1课作业完成

## 1. 大模型不是答案仓库

可以先用一个不完全准确、但好理解的比喻：

大模型像一个读过大量文字、非常擅长接话的人。它会根据当前上下文，预测接下来什么内容更合适。

它不是先去数据库找到一条“标准答案”，再原样返回。

因此同一个问题可能：

- 使用不同措辞；
- 强调不同重点；
- 产生质量差异；
- 在缺少信息时生成听起来合理但不真实的内容。

最后一种情况通常被称为“幻觉”。

## 2. 不稳定不等于没用

开放任务本来就可能有多个合理答案，例如：

> 帮我把这句话表达得更专业。

它没有唯一标准答案。模型的生成能力在这里很有价值。

真正的问题不是“输出必须每次完全一样”，而是：

- 是否保持事实；
- 是否满足目标；
- 是否符合格式；
- 是否越过安全边界；
- 是否能在失败时被系统识别。

## 3. 什么是 Temperature

现在只需要建立直觉：

- 较低：输出通常更稳定、保守；
- 较高：输出通常更多样、更有创造性；
- 它不能保证事实正确；
- 它也不是“答案质量”的旋钮。

代码生成、字段提取通常偏向稳定；创意文案可能允许更多变化。

## 4. 动手实验

向同一个大模型连续发送三次完全相同的内容：

```text
请将“这个需求根本做不了，你们先想清楚再说”改写为专业的职场沟通表达。
```

不要只判断“哪个好听”。请按四个标准检查：

| 标准 | 检查内容 |
| --- | --- |
| 原意保持 | 是否仍然表达当前需求存在实现问题？ |
| 事实可靠 | 是否增加了用户没有提供的原因或承诺？ |
| 语气合适 | 是否专业，但没有变得卑微？ |
| 可直接使用 | 是否可以直接发给同事或产品经理？ |

将三次结果和判断写入 `reviews/week-01.md`。

## 5. 今日验收

1. 为什么相同输入可能得到不同输出？
2. 输出不同是否一定代表模型失败？
3. 什么情况下输出虽然通顺，却不能用于产品？
4. Temperature 调低是否就能消除幻觉？

今天结束时，你应该开始形成一个习惯：不只看模型“能不能回答”，而要检查它在什么情况下会答错。
