# 第 9 周 · 第 3 天：动手实现核心能力

本周主题：从感觉到评测集

所属模块：AI 评测

建议时间：45～90 分钟

## 今日目标

完成核心实践：为职场助手建立第一版评测集。

## 1. 今日主任务

> 为职场助手建立第一版评测集

## 2. 实现步骤

1. 先定义输入、输出和失败状态；
2. 找到统一 `demo-app` 中最合适的实验目录；
3. 先完成最小成功路径；
4. 增加运行时校验；
5. 添加一个可观察信号，例如模型名、Token、引用、工具 Trace 或评分；
6. 保留现有 Mock 或固定样本，方便离线对照。

本模块的具体检查：

- 先写业务通过标准，再运行模型；
- 保存原始输出与校验后结果；
- 将格式失败和内容失败分开统计；
- 至少比较两个版本，避免单样本结论。

## 3. 实现约束

- 不重复创建独立项目；
- 不把 API Key 放到前端；
- 不猜测接口字段；
- 不为了“智能”取消服务端确定性规则；
- 一次只完成一个明确闭环。

## 4. 完成定义

今天不是以“代码写了很多”为完成，而是：

```text
一个明确输入
→ 经过本周新能力
→ 得到可验证输出
→ 失败时有明确状态
```

## 5. 代码讲解

完成后尝试不看代码讲清：

- 请求从哪个文件进入；
- 哪个函数最关键；
- 哪部分可以单元测试；
- 哪部分必须用模型评测。


## 结束条件

- 今天产生了可记录的答案、代码或测试证据；
- 能指出至少一个风险或限制；
- 没有用模型替代本应由程序控制的明确规则；
- 不清楚的接口字段已经停下来确认，没有自行猜测。
