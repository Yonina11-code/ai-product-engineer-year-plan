# 第 7 周 · 第 1 天：建立概念模型

本周主题：Prompt 变量与模板管理

所属模块：Prompt Engineering

建议时间：45～90 分钟

## 今日目标

建立模板变量、任务 Prompt 路由的正确直觉。

## 1. 从问题出发

本周要解决的工程问题是：**Prompt 变量与模板管理怎样让 AI 应用更可靠或更有价值？**

场景采用：职场沟通助手的 Prompt Registry。

Prompt 类似业务规则与函数入参的组合，但它不是严格执行的程序，所以必须配合测试集。

## 2. 今天的核心知识

### 1. 模板变量

模板变量 是本周链路中的一个关键决策点。理解时不要只背定义，要在“职场沟通助手的 Prompt Registry”中确认它接收什么输入、产生什么输出、失败时由谁负责。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

### 2. 任务 Prompt 路由

Prompt 是交给模型的完整任务上下文，包括固定规则、动态变量、用户输入和输出要求。它更像可评测的产品配置，不是神奇咒语。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

## 3. 画出位置

请把本周能力放进下面链路：

```text
用户问题 → 前端输入 → 服务端规则 → 模型/检索/工具
→ 结果校验 → 产品状态 → 用户反馈
```

重点标记：任务路由、变量来源和字段校验由代码决定；具体语言生成由模型完成。

## 4. 今日练习

用不超过200字解释本周主题，要求同时出现：

- 一个真实使用场景；
- 一个确定性程序职责；
- 一个模型不确定性；
- 一个失败风险。


## 结束条件

- 今天产生了可记录的答案、代码或测试证据；
- 能指出至少一个风险或限制；
- 没有用模型替代本应由程序控制的明确规则；
