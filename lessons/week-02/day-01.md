# 第 2 周 · 第 1 天：建立概念模型

本周主题：真实模型 API 与调用边界

所属模块：模型基础

建议时间：45～90 分钟

## 今日目标

建立messages 消息结构、API Key 与服务端环境变量的正确直觉。

## 1. 从问题出发

本周要解决的工程问题是：**真实模型 API 与调用边界怎样让 AI 应用更可靠或更有价值？**

场景采用：职场沟通助手调用 DeepSeek 的服务端链路。

把模型服务看作一个返回值不完全确定的外部 API：鉴权、超时和错误码仍是常规工程问题，但输出内容还需要额外评测。

## 2. 今天的核心知识

### 1. messages 消息结构

messages 用角色顺序表达系统规则、用户输入和历史对话。模型看到的是整个消息序列，因此历史内容的选择本身就是产品逻辑。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

### 2. API Key 与服务端环境变量

API Key 是模型服务的调用凭证，只能由服务端读取。它不是普通前端配置，泄露会造成越权调用和真实费用。

学习后请回答：

- 它解决什么问题？
- 它的输入与输出是什么？
- 哪种情况下不应该使用？

## 3. 画出位置

请把本周能力放进下面链路：

```text
用户问题 → 前端输入 → 服务端规则 → 模型/检索/工具
→ 结果校验 → 产品状态 → 用户反馈
```

重点标记：HTTP、环境变量、请求校验是确定性代码；自然语言生成、措辞选择和事实可靠性属于模型能力边界。
